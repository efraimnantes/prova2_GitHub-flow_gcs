# Testes simulados do dashboard EduStream 
 
Este arquivo registra os testes simulados realizados no Cenário 1 — GitHub Flow da Prova 2 de GCS. 
 
## Objetivo dos testes 
 
Verificar se as alterações feitas na homepage do dashboard EduStream não removeram funcionalidades existentes e se as novas seções foram adicionadas corretamente. 
 
## Casos de teste simulados 
 
| ID | Teste realizado | Resultado esperado | Status | 
|---|---|---|---| 
| TS-001 | Abrir o arquivo `edustream_dash2.0.html` no navegador | O dashboard deve carregar corretamente | Aprovado | 
| TS-002 | Verificar a seção "Visão Geral da Homepage" | A seção deve aparecer antes do rodapé | Aprovado | 
| TS-003 | Verificar a seção "Filtros do Dashboard" | A explicação dos filtros deve estar visível | Aprovado | 
| TS-004 | Verificar a seção "Indicadores de Medição" | A explicação dos indicadores deve estar visível | Aprovado | 
| TS-005 | Alterar o filtro de mês | Os dados do dashboard devem ser atualizados conforme o mês selecionado | Aprovado | 
| TS-006 | Alterar o filtro de squad | Os dados devem ser filtrados conforme o squad selecionado | Aprovado | 
| TS-007 | Clicar no botão "Limpar" | Os filtros devem voltar para a seleção inicial | Aprovado | 
| TS-008 | Verificar os gráficos | Os gráficos devem continuar sendo exibidos | Aprovado | 
| TS-009 | Verificar a tabela de métricas | A tabela deve continuar aparecendo com os registros detalhados | Aprovado | 
 
## Conclusão 
 
Os testes simulados indicam que as alterações feitas na homepage não quebraram o dashboard existente. As seções adicionadas foram exibidas corretamente e os filtros, gráficos e tabela permaneceram funcionais.
